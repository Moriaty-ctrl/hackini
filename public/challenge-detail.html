Overview

You've been hired to perform a security assessment of a new e-commerce platform before it goes live. The platform includes user authentication, product browsing, shopping cart functionality, and a payment processing system.

Your goal is to identify and exploit vulnerabilities in the application, with a focus on the payment gateway integration and authentication mechanisms.

Objectives

  • Find and exploit authentication bypass vulnerabilities
  • Identify vulnerabilities in the payment processing system
  • Access administrative functions without proper authorization
  • Extract sensitive customer information from the database
  • Manipulate product prices during checkout

Environment

You'll be provided with access to a virtual environment containing the e-commerce application. The application is built using a modern web stack:

  • Frontend: React.js
  • Backend: Node.js with Express
  • Database: MongoDB
  • Payment Processing: Stripe API integration

The application is accessible at the following URL once you start the challenge:

https://ecommerce-challenge.cyberdojo.io/[your-session-id]

Getting Started

Once you start the challenge, you'll receive credentials for a basic user account:

Username: user@example.com
Password: password123

You can use these credentials to log in and explore the application. Your goal is to escalate your privileges and find ways to exploit the system.

Here's a basic command to test the API endpoint:

$ curl -X GET https://ecommerce-challenge.cyberdojo.io/[your-session-id]/api/products
[ { "id": "prod_001", "name": "Premium Headphones", "price": 199.99, "description": "High-quality wireless headphones with noise cancellation" }, { "id": "prod_002", "name": "Smartphone Case", "price": 29.99, "description": "Durable protective case for smartphones" }, ... ]

Submission

To complete this challenge, you need to:

  1. Find at least 3 distinct vulnerabilities in the application
  2. Document your findings, including the vulnerability type, impact, and exploitation method
  3. Provide proof of successful exploitation (screenshots, code snippets, etc.)
  4. Submit your findings through the challenge submission form

Your submission will be reviewed by our team, and points will be awarded based on the severity of the vulnerabilities you discover and the quality of your documentation.

Hints

Authentication Hint
25 points
Check for insecure JWT implementation and token validation issues. Look for ways to manipulate or forge tokens.
Payment Processing Hint
50 points
The payment processing system might not properly validate the price data sent from the client. Try intercepting and modifying requests during checkout.
Admin Access Hint
75 points
The application might have hidden admin endpoints that are not properly secured. Try directory enumeration and check for common admin paths.
